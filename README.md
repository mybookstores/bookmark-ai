# BookmarkAI

> 一个更聪明的浏览器书签插件：保存网页、自动总结、分类整理、全文检索，让收藏真正可回顾、可利用。

BookmarkAI 是一个本地优先的 Chrome 扩展。它会在你保存网页时自动提取标题、正文片段、站点图标与预览图，并可选调用 AI 生成中文摘要。你可以用文件夹、标签、搜索和详情预览管理收藏内容，也可以直接通过右键菜单快速保存页面或链接。

## 为什么做它

传统浏览器收藏夹最大的问题不是“存不下”，而是“找不到、看不懂、懒得回看”。

BookmarkAI 的目标是把书签从“死链接列表”变成一个轻量的个人知识入口：

- 保存时自动生成上下文，而不是只存一个标题
- 用摘要帮助你快速回忆内容
- 用文件夹和标签降低整理成本
- 用搜索和详情预览降低再次使用成本

## 核心卖点

- **一键保存网页**：自动提取标题、正文片段、favicon、OG 图片
- **AI 中文摘要**：支持 OpenAI / Anthropic / Ollama / 第三方 OpenAI 兼容接口
- **AI 自动标签**：保存时自动生成相关标签
- **本地优先**：所有书签与设置保存在本地 IndexedDB，无需账号
- **详情预览**：先看摘要、摘录、链接和标签，再决定是否打开原网页
- **右键菜单保存**：支持”保存当前页面”、”保存链接”、”保存选中文本”
- **文件夹 + 标签**：同时支持主题分组和多维筛选
- **全文搜索**：按标题、URL、摘要、标签快速检索
- **书签排序**：按时间、标题、域名排序
- **个人笔记**：为每个书签添加个人笔记和感想
- **书签分享**：生成分享卡片图片、一键复制链接
- **多格式导入导出**：支持 JSON / Markdown / CSV / Chrome 书签 HTML
- **标签管理**：统一管理标签（重命名、删除）
- **深色模式**：支持浅色 / 深色 / 跟随系统

## 安装方式

### 1. 克隆项目

```bash
git clone https://github.com/mybookstores/bookmark-ai.git
cd bookmark-ai
```

### 2. 安装依赖

```bash
pnpm install
```

### 3. 构建扩展

```bash
pnpm build
```

### 4. 加载到 Chrome

1. 打开 `chrome://extensions/`
2. 开启右上角 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择项目下的 `dist/` 目录

## 使用说明

### 保存当前页面

1. 打开任意网页
2. 点击扩展图标
3. 点击 **保存当前页面**
4. 若已配置 API Key，BookmarkAI 会自动生成中文摘要

### 右键快速保存

安装扩展后，可以直接：

- 在页面空白处右键 → **保存当前页面到 BookmarkAI**
- 在链接上右键 → **保存链接到 BookmarkAI**

### 查看详情

- 普通点击书签卡片：打开详情预览层
- 点击卡片右上角外链按钮：直接打开原网页

### 快捷键

- `Cmd/Ctrl + S`：保存当前页面
- `Cmd/Ctrl + F`：聚焦搜索框
- `Esc`：关闭详情预览层

## AI 配置

### 支持的提供商

- OpenAI
- Anthropic Claude
- Ollama
- 自定义 OpenAI 兼容 API

### 第三方兼容 API 示例

```txt
Base URL: https://lanyiapi.com/v1
Model:    claude-sonnet-4-20250514
```

在设置页中：

1. 选择 **自定义 API**
2. 填入 Base URL
3. 填入 API Key
4. 填入模型名
5. 保存即可

> API Key 仅保存在本地浏览器中，不会上传到项目后端，因为项目没有后端。

## 项目结构

```bash
bookmark-ai/
├── dist/                     # Chrome 加载目录
├── icons/                    # 扩展图标
│   └── source/               # 图标源文件
├── src/
│   ├── background/           # MV3 service worker / 右键菜单
│   ├── components/           # UI 组件
│   ├── lib/                  # AI、存储、共享保存逻辑
│   ├── popup/                # 扩展弹窗入口
│   ├── stores/               # Zustand 状态管理
│   └── types/                # 类型定义
├── manifest.json
├── popup.html
└── README.md
```

## 关键实现点

- **MV3 扩展结构**：popup + service worker
- **共享保存链路**：popup 保存和右键菜单保存走同一套逻辑
- **本地优先架构**：无需后端即可完整运行
- **OpenAI 兼容接口支持**：便于接入第三方网关
